# 2D Graphics Editor (Mini-Project in C)

A structured, modular, terminal-based vector and raster graphics editor written in standard C (C99). The project allows users to draw, delete, and modify shapes (rectangles, horizontal lines, right-angled triangles, and circles) on a 20x40 character canvas.

---

## 1. Project Objectives
* **Modular Software Design:** Separating shape model representations, input parsing, coordinate boundaries validation, and canvas rendering into isolated functions.
* **Structured Data Models:** Using `struct`, `union`, and `enum` structures in C to store and handle heterogeneous geometric attributes.
* **Canvas State Rebuilding:** Demonstrating clean MVC-like behavior where a central model (an array of objects) determines the visual matrix representation rather than raw canvas pixels.
* **Bounds Verification:** Ensuring zero buffer out-of-bound errors (e.g. segmentation faults) by validating all geometric shapes prior to their placement.

---

## 2. Features
1. **Interactive Console Menu:** Simple keyboard inputs let users run canvas procedures.
2. **Dynamic 2D Canvas (20x40):** Auto-renders with grid rulers (0-9 columns on top, 0-19 rows on the left) for accurate coordinate tracking.
3. **Structured Shape Drawings:**
   * **Rectangle:** Custom top-left point, width, and height. Draws outline using `*`.
   * **Line:** Custom horizontal start point and length.
   * **Triangle:** Right-angled triangle originating at top-left vertex, extending down and right.
   * **Circle:** Mathematical circle rendering based on Euclidean distances.
4. **Interactive Deletion:** Users enter coordinates `(x, y)` to delete the shape overlapping that location. Remapped canvas is instantly displayed.
5. **Interactive Modification:** Select shapes by entering overlap coordinates and move them to new coordinate positions.
6. **Automatic Compaction:** Cleans deleted shape slots from tracking arrays to avoid buffer limits.
7. **Reset:** Wipe canvas and clear all drawn objects instantly.

## 3. Data Structures Used

```c
/* Shape type indicators */
typedef enum {
    RECTANGLE = 1,
    LINE = 2,
    TRIANGLE = 3,
    CIRCLE = 4
} ShapeType;

/* Individual structures mapping geometric specifications */
typedef struct { int x, y, width, height; } Rectangle;
typedef struct { int x, y, length; } Line;
typedef struct { int x, y, height; } Triangle;
typedef struct { int cx, cy, r; } Circle;

/* High-level wrapper object representing any active/deleted shape */
typedef struct {
    int id;               /* Unique Identifier */
    ShapeType type;       /* Shape type indicator */
    int isActive;         /* Visible status: 1 = Active, 0 = Inactive (Deleted) */
    union {
        Rectangle rect;
        Line line;
        Triangle tri;
        Circle circ;
    } data;               /* Union space optimizing memory storage */
} GraphicObject;
```

## 4. Algorithms Used

### 4.1 Circle Bounding Area Equation
The editor maps circular pixels by applying the standard algebraic equation:
$$(x - c_x)^2 + (y - c_y)^2 \le r^2$$
For every coordinate point $(x, y)$ on the grid, if the sum of squares of offsets from the center $(c_x, c_y)$ is less than or equal to $r^2$, the character `*` is printed, rendering a filled, mathematically accurate circle on a raster layout.

### 4.2 Right-Angled Triangle Mapping
Triangles are drawn moving downwards. For a top-left offset point $(x, y)$ and height $h$:
* For row index $i$ from $0$ to $h-1$:
  * Fill column index $j$ from $x$ to $x + i$ at row $y + i$ with `*`.

### 4.3 Z-Order Object Intersection Detection
When a user enters coordinate point $(p_x, p_y)$ for modification or deletion:
* The editor searches the tracking array in reverse order (newest to oldest).
* Points are assessed geometrically against active objects:
  * **Rectangle:** $p_x \in [x, x + \text{width}) \land p_y \in [y, y + \text{height})$
  * **Line:** $p_y = y \land p_x \in [x, x + \text{length})$
  * **Triangle:** $p_y \in [y, y + h) \land p_x \in [x, x + (p_y - y)]$
  * **Circle:** $(p_x - c_x)^2 + (p_y - c_y)^2 \le r^2$
* The topmost shape detected is returned first.

## 5. Compilation and Execution Steps

### 5.1 Compiling in Code::Blocks
1. Open Code::Blocks IDE.
2. Select **File > New > Project...** and choose **Console Application**, click **Next**.
3. Choose **C** language and click **Next**.
4. Set the project title to `2D Graphics Editor` and specify your folder path.
5. In the next steps, select standard compilers (e.g. GCC) and click **Finish**.
6. Replace the autogenerated `main.c` content with our `main.c` code.
7. Click **Build and Run** (or press `F9`) on your toolbar.

### 5.2 Compiling with Command Line GCC (Alternative)
Run the following compiler command in PowerShell or Command Prompt:
```bash
gcc -Wall -Wextra -std=c99 main.c -o graphics_editor.exe
.\graphics_editor.exe
```

## 6. Sample User Execution Steps
1. **Draw a Rectangle:**
   * Select option `1`.
   * Inputs: X = `5`, Y = `2`, Width = `12`, Height = `6`.
   * Result: A $12 \times 6$ rectangular outline is drawn.
2. **Draw a Circle:**
   * Select option `4`.
   * Inputs: Center X = `25`, Center Y = `10`, Radius = `4`.
   * Result: A circle of radius 4 appears.
3. **Move a Shape (Modify):**
   * Select option `6`.
   * Select coordinates to target (e.g., center of circle: `25`, `10`).
   * Enter new coordinates: X = `15`, Y = `8`.
   * Result: The circle disappears from `(25,10)` and is redrawn at center `(15,8)`.
4. **Delete a Shape:**
   * Select option `5`.
   * Enter a coordinate intersecting the rectangle (e.g., `5`, `2`).
   * Result: The rectangle is removed from the screen, and the circle remains.
